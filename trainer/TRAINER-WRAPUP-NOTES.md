# Trainer Wrap-Up Notes

Facilitator-only material. Do not reference this file from participant agents.

## Goal

Reinforce the learning and collect questions.

## Discussion Prompts

- What did you describe that made the app better?
- What did the first version get right?
- What did you improve after reviewing it?
- What made the result feel app-like rather than report-like?
- Where could Describe -> Review -> Improve help in customer work?

## Key Messages

- The app was not the main point; the collaboration pattern was.
- A clear spec improves the first result.
- Reviewing output is part of the work.
- Small improvement requests are powerful.
- Grounding with trusted data and design sources improves consistency.

## Q&A Notes

If asked about MCP:
"MCP is a way for the assistant to connect to trusted tools and data sources."

If asked about skills:
"Skills are reusable instructions that apply patterns automatically."

If asked about agents:
"Agents are configured assistants with a specific role and boundaries."

If asked whether this replaces Microsoft 365 Copilot documents:
"No. This workshop shows a different value: interactive app-like experiences where users can explore, filter, score, and decide."
